What is JavaScript?

JavaScript is the programming language of the web. While HTML provides structure and CSS provides styling, JavaScript makes web pages interactive. It can respond to user actions, manipulate content, validate forms, create animations, and communicate with servers.

üí° Key Insight

JavaScript runs in the browser (client-side) and can also run on servers (Node.js). For building websites like PassEPPP, we'll focus on client-side JavaScript that makes web pages dynamic and interactive.

Adding JavaScript to HTML

There are three ways to add JavaScript to your web pages:

1. Internal Script (In the HTML file)

HTML
<!-- Place at the end of <body> for best performance --> <script> console.log('Hello from JavaScript!'); </script> </body>

2. External Script (Separate .js file) ‚Äî Recommended!

HTML
<!-- Link to external JavaScript file --> <script src="js/main.js"></script> </body>

3. Inline JavaScript (Avoid this!)

HTML (Not Recommended)
<!-- Hard to maintain, mix of concerns --> <button onclick="alert('Clicked!')">Click Me</button>
‚úÖ Best Practice

Always use external JavaScript files. This separates your code, makes it reusable, easier to debug, and can be cached by the browser.

Variables: Storing Data

Variables are containers for storing data. In modern JavaScript, we use let and const to declare variables.

JavaScript
// const - for values that won't change (constants) const appName = 'PassEPPP'; const maxQuestions = 175; const isExamDay = true; // let - for values that will change let currentScore = 0; let userName = 'Student'; let questionsAnswered = 0; // Updating let variables currentScore = 85; // ‚úÖ This works questionsAnswered = questionsAnswered + 1; // ‚úÖ Now it's 1 // You CANNOT reassign const // appName = 'NewName'; // ‚ùå Error!
‚ö†Ô∏è Avoid var

You may see var in older code. It has scoping issues that can cause bugs. Always use const by default, and let when you need to reassign.

Naming Convention

JavaScript uses camelCase for variable names:

JavaScript
// ‚úÖ Good variable names (camelCase) let firstName = 'John'; let totalQuestions = 50; let isLoggedIn = true; let userEmailAddress = 'test@example.com'; // ‚ùå Avoid these naming styles // let first_name = 'John'; // snake_case (Python style) // let FirstName = 'John'; // PascalCase (for classes) // let FIRSTNAME = 'John'; // ALL_CAPS (for true constants)

Data Types

JavaScript has several fundamental data types. Here are the most common ones:

Type Description Examples
String Text wrapped in quotes 'Hello', "World", `Template`
Number Integers or decimals 42, 3.14, -10
Boolean True or false values true, false
Undefined Variable declared but not assigned undefined
Null Intentionally empty value null
Array Ordered list of values [1, 2, 3], ['a', 'b']
Object Collection of key-value pairs { name: 'John', age: 25 }
JavaScript - Data Types Examples
// Strings - text data const greeting = 'Hello, World!'; const courseName = "EPPP Exam Prep"; // Numbers - no quotes needed const totalQuestions = 175; const passingScore = 70.5; // Booleans - true or false const isLoggedIn = true; const hasCompletedQuiz = false; // Arrays - lists of items const topics = ['Ethics', 'Assessment', 'Treatment']; const scores = [85, 92, 78, 95]; // Objects - structured data const user = { name: 'Ethan', email: 'ethan@example.com', score: 85, isPremium: true }; // Checking types with typeof console.log(typeof greeting); // "string" console.log(typeof totalQuestions); // "number" console.log(typeof isLoggedIn); // "boolean"

Template Literals

Template literals (using backticks ` `) are the modern way to work with strings. They allow you to embed variables and expressions directly:

JavaScript
const userName = 'Ethan'; const score = 85; const total = 100; // Old way - concatenation with + const oldMessage = 'Hello, ' + userName + '! Your score is ' + score + '/' + total; // Modern way - template literals with ${} const newMessage = `Hello, ${userName}! Your score is ${score}/${total}`; // You can use expressions inside ${} const percentage = `You scored ${(score / total) * 100}%`; // Multi-line strings are easy with template literals const html = ` <div class="card"> <h2>${userName}</h2> <p>Score: ${score}</p> </div> `;

The Console: Your Debugging Friend

The browser console is essential for debugging. Use console.log() to output values and check your code:

JavaScript
// Basic logging console.log('Hello!'); // Simple message console.log('Score:', 85); // Multiple values console.log('User:', { name: 'Ethan' }); // Objects // Useful console methods console.warn('This is a warning!'); // Yellow warning console.error('This is an error!'); // Red error console.table(['a', 'b', 'c']); // Display as table // Debugging with labels const user = { name: 'Ethan', score: 85 }; console.log('User data:', user); // Labeled output

Basic Operators

Arithmetic Operators

JavaScript
let a = 10; let b = 3; console.log(a + b); // 13 - Addition console.log(a - b); // 7 - Subtraction console.log(a * b); // 30 - Multiplication console.log(a / b); // 3.333... - Division console.log(a % b); // 1 - Remainder (modulo) console.log(a ** b); // 1000 - Exponent (10^3) // Increment and decrement let count = 0; count++; // count is now 1 count--; // count is now 0 // Shorthand assignment let score = 50; score += 10; // Same as: score = score + 10 (now 60) score -= 5; // Same as: score = score - 5 (now 55)

Comparison Operators

JavaScript
const x = 10; const y = '10'; // Comparison returns true or false console.log(x === 10); // true - Strict equality (recommended!) console.log(x === y); // false - Different types (number vs string) console.log(x == y); // true - Loose equality (avoid!) console.log(x !== 5); // true - Not equal console.log(x > 5); // true - Greater than console.log(x < 5); // false - Less than console.log(x >= 10); // true - Greater than or equal console.log(x <= 10); // true - Less than or equal
‚ö†Ô∏è Always Use === (Strict Equality)

The == operator does type coercion which can lead to unexpected bugs. Always use === for comparisons to avoid surprises like '10' == 10 returning true.

Conditional Statements

Use if, else if, and else to make decisions in your code:

JavaScript
const score = 85; if (score >= 90) { console.log('Excellent! You got an A'); } else if (score >= 80) { console.log('Great job! You got a B'); } else if (score >= 70) { console.log('Good work! You got a C'); } else { console.log('Keep studying!'); } // Logical operators for complex conditions const isLoggedIn = true; const isPremium = true; // AND (&&) - both must be true if (isLoggedIn && isPremium) { console.log('Welcome, premium member!'); } // OR (||) - at least one must be true if (score >= 90 || isPremium) { console.log('You get bonus content!'); } // NOT (!) - inverts boolean if (!isLoggedIn) { console.log('Please log in'); }

Key Takeaways

  • Use const by default, let when you need to reassign
  • JavaScript has primitive types (string, number, boolean) and complex types (arrays, objects)
  • Template literals (`${variable}`) are the modern way to build strings
  • console.log() is your best friend for debugging
  • Always use === for comparisons, never ==
  • Use if/else statements to control program flow
