Responsive Design & CSS Variables

45 min Lesson 10 of 28
What you'll learn
  • Use media queries for different screen sizes
  • Implement mobile-first design approach
  • Create and use CSS custom properties (variables)

Why Responsive Design?

Over 50% of web traffic comes from mobile devices. Your website MUST work on all screen sizes — phones, tablets, laptops, and desktops.

Responsive design means your layout adapts to the user's screen. You achieve this with:

  • Fluid layouts (percentages, fr units, flexbox, grid)
  • Flexible images
  • Media queries for specific breakpoints

The Viewport Meta Tag

This tag is essential for mobile responsiveness. Without it, mobile browsers render at desktop width and zoom out.

<!-- ALWAYS include this in your <head> -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries

Media queries let you apply CSS rules only at certain screen widths:

responsive.css 
/* Base styles (mobile-first) */
.container {
  padding: 16px;
}

/* Tablet and up */
@media (min-width: 768px) {
  .container {
    padding: 24px;
  }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .container {
    padding: 32px;
    max-width: 1200px;
    margin: 0 auto;
  }
}

Common Breakpoints

Breakpoint Target Media Query
Small Phones @media (min-width: 640px)
Medium Tablets @media (min-width: 768px)
Large Laptops @media (min-width: 1024px)
XL Desktops @media (min-width: 1280px)
Mobile-First Approach

Write your base CSS for mobile, then use min-width media queries to add styles for larger screens. This ensures your mobile experience is solid and larger screens get enhancements.

Responsive Navigation Pattern

nav-responsive.css 
/* Mobile: Stack nav links vertically, hide by default */
.nav-links {
  display: none;
  flex-direction: column;
  width: 100%;
}

.nav-links.active {
  display: flex;
}

.hamburger {
  display: block;  /* Show hamburger on mobile */
}

/* Desktop: Show links in a row, hide hamburger */
@media (min-width: 768px) {
  .nav-links {
    display: flex;
    flex-direction: row;
    width: auto;
    gap: 24px;
  }

  .hamburger {
    display: none;
  }
}

CSS Custom Properties (Variables)

CSS variables let you define reusable values. They're incredibly powerful for theming and consistency.

variables.css 
/* Define variables in :root (global scope) */
:root {
  --color-primary: #3b82f6;
  --color-secondary: #a855f7;
  --color-text: #fafafa;
  --color-text-muted: #a1a1aa;
  --color-bg: #09090b;

  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;

  --radius: 8px;
  --font-body: 'Inter', sans-serif;
}

/* Use variables with var() */
.button {
  background: var(--color-primary);
  padding: var(--spacing-sm) var(--spacing-md);
  border-radius: var(--radius);
  font-family: var(--font-body);
}

.card {
  background: var(--color-bg);
  padding: var(--spacing-lg);
  border-radius: var(--radius);
}

Why Use CSS Variables?

  • Consistency: Change a color in one place, updates everywhere
  • Theming: Easy dark/light mode switching
  • Readability: var(--color-primary) is clearer than #3b82f6
  • JavaScript access: You can read/write CSS variables from JS

Dark Mode Example

:root {
  --bg: #ffffff;
  --text: #1a1a1a;
}

@media (prefers-color-scheme: dark) {
  :root {
    --bg: #09090b;
    --text: #fafafa;
  }
}

body {
  background: var(--bg);
  color: var(--text);
}
Today's Project: Make Your Landing Page Mobile-Responsive
  1. Add CSS variables for all your colors, spacing, and fonts at the top of your stylesheet
  2. Replace all hardcoded values with var(--name)
  3. Add a mobile navigation with hamburger menu (hidden on desktop)
  4. Stack your feature cards vertically on mobile, grid on desktop
  5. Adjust font sizes and padding for mobile
  6. Test by resizing your browser window

Key Takeaways

  • Mobile-first: Base styles for mobile, media queries for larger screens
  • min-width queries: Apply styles at this width and above
  • CSS variables: Define in :root, use with var()
  • Always include the viewport meta tag!
