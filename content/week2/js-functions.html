What are Functions?

Functions are reusable blocks of code that perform specific tasks. Instead of writing the same code over and over, you define it once in a function and call it whenever needed. Functions are the building blocks of JavaScript applications.

üí° Think of Functions Like Recipes

A recipe (function) has a name, takes ingredients (parameters), follows steps (code), and produces a dish (return value). Once you have the recipe, you can make the dish anytime by following it.

Function Declaration

The most common way to create a function:

JavaScript
// Function declaration syntax function greet() { console.log('Hello, World!'); } // Calling (invoking) the function greet(); // Output: Hello, World! greet(); // Output: Hello, World! // Function with parameters (inputs) function greetUser(name) { console.log(`Hello, ${name}!`); } greetUser('Ethan'); // Output: Hello, Ethan! greetUser('Sarah'); // Output: Hello, Sarah! // Multiple parameters function addNumbers(a, b) { console.log(a + b); } addNumbers(5, 3); // Output: 8

Return Values

Functions can return values using the return keyword. This lets you use the result elsewhere in your code:

JavaScript
// Function that returns a value function add(a, b) { return a + b; } // Store the returned value in a variable const sum = add(10, 5); console.log(sum); // Output: 15 // Use returned value directly console.log(add(3, 7)); // Output: 10 // Use in calculations const total = add(10, 20) + add(5, 5); // 30 + 10 = 40 // Practical example: Calculate quiz score percentage function calculatePercentage(correct, total) { return (correct / total) * 100; } const myScore = calculatePercentage(17, 20); console.log(`Your score: ${myScore}%`); // Output: Your score: 85%
‚ö†Ô∏è Return Stops Execution

Once a return statement executes, the function stops immediately. Any code after return will never run.

JavaScript
function checkScore(score) { if (score >= 70) { return 'Pass'; // Function exits here if true } return 'Fail'; // Only runs if score < 70 } console.log(checkScore(85)); // Output: Pass console.log(checkScore(60)); // Output: Fail

Default Parameters

You can set default values for parameters. If no argument is passed, the default is used:

JavaScript
// Default parameter values function greet(name = 'Guest', greeting = 'Hello') { return `${greeting}, ${name}!`; } console.log(greet()); // Hello, Guest! console.log(greet('Ethan')); // Hello, Ethan! console.log(greet('Ethan', 'Welcome')); // Welcome, Ethan! // Practical: Quiz settings with defaults function createQuiz(topic, questions = 10, timeLimit = 30) { return { topic: topic, questions: questions, timeLimit: timeLimit }; } const quiz1 = createQuiz('Ethics'); // 10 questions, 30 min const quiz2 = createQuiz('Assessment', 20); // 20 questions, 30 min

Arrow Functions

Arrow functions are a shorter, modern syntax for writing functions. They're especially useful for simple operations:

JavaScript
// Traditional function function add(a, b) { return a + b; } // Arrow function (equivalent) const addArrow = (a, b) => { return a + b; }; // Short arrow function (implicit return) // When there's only one expression, you can omit {} and return const addShort = (a, b) => a + b; // Single parameter - parentheses optional const double = n => n * 2; const square = n => n * n; console.log(double(5)); // 10 console.log(square(4)); // 16 // No parameters - empty parentheses required const sayHello = () => 'Hello!'; const getTimestamp = () => new Date().toISOString();

Traditional Function

Use when you need a named function, hoisting behavior, or this binding.

function name() { }

Arrow Function

Use for callbacks, short operations, and when you want concise syntax.

const name = () => { }
‚úÖ When to Use Arrow Functions
  • Array methods like map, filter, forEach
  • Simple one-liner operations
  • Callback functions
  • When you don't need your own this context

Practical Examples

Let's look at real-world functions you might use when building PassEPPP:

// Calculate letter grade from percentage function getLetterGrade(percentage) { if (percentage >= 90) return 'A'; if (percentage >= 80) return 'B'; if (percentage >= 70) return 'C'; if (percentage >= 60) return 'D'; return 'F'; } // Format time remaining function formatTime(seconds) { const mins = Math.floor(seconds / 60); const secs = seconds % 60; return `${mins}:${secs.toString().padStart(2, '0')}`; } console.log(formatTime(125)); // "2:05" console.log(formatTime(60)); // "1:00" // Generate progress message function getProgressMessage(completed, total) { const percentage = Math.round((completed / total) * 100); if (percentage === 100) { return 'üéâ Congratulations! Functions Calling Functions

Functions can call other functions, allowing you to build complex logic from simple pieces:

JavaScript
// Small, focused functions const calculatePercentage = (correct, total) => (correct / total) * 100; const getLetterGrade = (percentage) => { if (percentage >= 90) return 'A'; if (percentage >= 80) return 'B'; if (percentage >= 70) return 'C'; if (percentage >= 60) return 'D'; return 'F'; }; const isPassing = (percentage) => percentage >= 70; // Main function that uses the smaller ones function getQuizResults(correct, total) { const percentage = calculatePercentage(correct, total); const grade = getLetterGrade(percentage); const passed = isPassing(percentage); return { score: `${correct}/${total}`, percentage: `${percentage.toFixed(1)}%`, grade: grade, passed: passed, message: passed ? 'Great job!' : 'Keep studying!' }; } const results = getQuizResults(17, 20); console.log(results); // { // score: "17/20", // percentage: "85.0%", // grade: "B", // passed: true, // message: "Great job!" // }

Key Takeaways

  • Functions are reusable blocks of code - define once, use many times
  • Parameters allow functions to accept inputs; return values provide outputs
  • Default parameters provide fallback values when arguments aren't passed
  • Arrow functions offer concise syntax: const fn = (x) => x * 2
  • Build complex logic by having functions call other functions
  • Keep functions small and focused on a single task
