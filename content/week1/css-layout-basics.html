The Display Property

Every HTML element has a default display value that determines how it behaves in the page flow. Understanding this is crucial for layout.

Block Elements

Block elements take up the full width available and start on a new line.

/* Block elements: div, p, h1-h6, section, article, header, footer, etc. */

.block-example {
  display: block;
  width: 100%;  /* Takes full width by default */
}

Inline Elements

Inline elements only take up as much width as needed and don't break the line. They flow with text.

/* Inline elements: span, a, strong, em, img, etc. */

.inline-example {
  display: inline;
}

/* WARNING: width and height have NO effect on inline elements! */

Inline-Block: Best of Both

Inline-block elements flow inline BUT you can set width and height!

inline-block.css 
.button {
  display: inline-block;
  padding: 12px 24px;
  width: 150px;       /* Works! */
  text-align: center;
}

/* Multiple buttons will sit side-by-side */

Common Pattern

Navigation links are often styled with display: inline-block so they sit side-by-side but can have padding, width, and height.

The Position Property

Position controls HOW an element is placed on the page. Combined with top, right, bottom, left, it gives you precise control.

position: static (Default)

Normal document flow. top/right/bottom/left have no effect.

position: relative

Positioned relative to its normal position. The original space is preserved.

.nudged {
  position: relative;
  top: 10px;    /* Moves DOWN 10px from normal position */
  left: 20px;   /* Moves RIGHT 20px from normal position */
}

position: absolute

Removed from normal flow. Positioned relative to its nearest positioned ancestor (or the viewport if none).

absolute.css 
/* Parent needs position: relative to contain the absolute child */
.parent {
  position: relative;
}

.badge {
  position: absolute;
  top: -8px;
  right: -8px;
  background: red;
  border-radius: 50%;
  width: 24px;
  height: 24px;
}
The Parent Trap

Absolute elements look for a positioned parent. If none exists, they position relative to the viewport. Always set position: relative on the parent you want to contain absolute children!

position: fixed

Positioned relative to the viewport. Stays in place when scrolling.

fixed-header.css 
.fixed-header {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;         /* Stretch full width */
  height: 64px;
  background: #09090b;
  z-index: 100;    /* Stay above other content */
}

/* Don't forget to add padding to body so content isn't hidden! */
body {
  padding-top: 64px;
}

position: sticky

Hybrid: acts like relative until you scroll past it, then becomes fixed.

.sticky-sidebar {
  position: sticky;
  top: 20px;  /* Sticks 20px from top when scrolling */
}

Z-Index: Stacking Order

When elements overlap, z-index controls which one appears on top. Higher values = closer to viewer.

/* Only works on positioned elements (not static) */
.behind  { position: relative; z-index: 1; }
.in-front { position: relative; z-index: 10; }
.modal    { position: fixed; z-index: 1000; }  /* Above everything */
Today's Project: Fixed Navigation Header
  1. Create a <header> with:
    • A logo/site name on the left
    • Navigation links on the right
  2. Style it with position: fixed at the top
  3. Set left: 0; right: 0; to stretch full width
  4. Add a background color and subtle border-bottom
  5. Set z-index: 100 to stay above content
  6. Add padding-top to your body equal to the header height
  7. Make nav links display: inline-block with padding

Key Takeaways

  • display: block — full width, new line
  • display: inline — flows with text, can't set width/height
  • display: inline-block — best of both
  • position: fixed — stays in viewport (great for headers)
  • position: absolute — needs a positioned parent
