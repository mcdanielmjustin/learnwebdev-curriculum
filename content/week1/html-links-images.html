Links, Images & Semantic HTML

45 min Lesson 3 of 28
What you'll learn
  • Create links between pages and to external sites
  • Add images with proper attributes
  • Use semantic HTML tags for better structure

Links: The Web's Superpower

Links are what make the web a web. They connect pages together, creating the interconnected experience we all use every day. The anchor tag <a> creates links.

Basic Link Syntax

links.html 
<!-- Link to another website -->
<a href="https://google.com">Go to Google</a>

<!-- Link to another page on your site -->
<a href="about.html">About Us</a>

<!-- Link to a section on the same page -->
<a href="#contact">Jump to Contact</a>

<!-- Link that opens in new tab -->
<a href="https://github.com" target="_blank">GitHub (new tab)</a>
The href Attribute

href stands for "hypertext reference" — it tells the browser where to go when clicked. It can be a full URL, a relative path, or an ID reference (#).

Link Paths Explained

  • Absolute URLs: https://example.com/page — full address, used for external sites
  • Relative paths: about.html or pages/contact.html — relative to current file
  • Root-relative: /pages/about.html — starts from website root
  • Anchor links: #section-id — jumps to an element with that ID

Images

The <img> tag embeds images. It's a self-closing tag — no closing tag needed!

images.html 
<!-- Basic image -->
<img src="photo.jpg" alt="A beautiful sunset">

<!-- Image with size constraints -->
<img src="logo.png" alt="Company Logo" width="200">

<!-- Image from the web -->
<img src="https://example.com/image.jpg" alt="Description">
Always Use Alt Text!

The alt attribute is required for accessibility. Screen readers read it aloud for visually impaired users. It also shows if the image fails to load. Write descriptive alt text!

Image Best Practices

  • Optimize file size — large images slow down your site
  • Use descriptive filenames — team-photo.jpg not IMG_12345.jpg
  • Choose the right format — JPG for photos, PNG for graphics with transparency, SVG for icons
  • Set dimensions — prevents layout shift while loading

Semantic HTML

Semantic tags describe the meaning of content, not just how it looks. This helps browsers, search engines, and screen readers understand your page structure.

The Old Way (Don't Do This)

<!-- Non-semantic: divs everywhere! -->
<div class="header">...</div>
<div class="navigation">...</div>
<div class="main-content">...</div>
<div class="footer">...</div>

The Right Way (Semantic HTML5)

semantic.html 
<header>
  <nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
    <a href="/contact">Contact</a>
  </nav>
</header>

<main>
  <article>
    <h1>Article Title</h1>
    <p>Article content...</p>
  </article>

  <aside>
    <h2>Related Links</h2>
    <!-- Sidebar content -->
  </aside>
</main>

<footer>
  <p>© 2026 My Website</p>
</footer>

Key Semantic Tags

  • <header> — Top section, usually contains logo and navigation
  • <nav> — Navigation links
  • <main> — The main content (only one per page!)
  • <article> — Self-contained content (blog post, news article)
  • <section> — Thematic grouping of content
  • <aside> — Sidebar, related content
  • <footer> — Bottom section, copyright, links
Why Semantic HTML Matters

SEO: Search engines understand your content better. Accessibility: Screen readers can navigate by landmarks. Maintainability: Code is easier to read and understand.

Today's Project: Multi-Page Portfolio

Create a 3-page portfolio site structure:

  1. Create three files: index.html, projects.html, contact.html
  2. Each page should have the same <header> with <nav> linking to all three pages
  3. Use <main> for unique content on each page
  4. Add a <footer> with your name and copyright
  5. On the projects page, add at least 2 images with proper alt text
  6. On the contact page, add a link to your email: href="mailto:you@email.com"

Key Takeaways

  • Links use <a href=""> — connect pages and the web together
  • Images need alt text — for accessibility and SEO
  • Semantic tags describe meaning — header, nav, main, article, footer
  • Structure matters — good HTML makes everything else easier
